# Changelog

## [2.0.0] — 2026-05-11

### Added
- **PROMPTS.md**: 子代理提示词模板独立文件，4 套模板（开场轮、交锋轮、迟到者、单人回应）
- **前置自检机制**: 子代理发言前内部回答 3 个问题，防止跑题和表演式发言
- **50字摘要机制**: 每轮发言末尾输出 `【摘要】`，后续轮次传摘要不传全文，大幅降低 token 消耗
- **三方批评自检**: 整合方案必须经过三个参会者的「反事实批评」验证
- **EXAMPLE.md**: 完整的圆桌会议案例演示（35岁中台经理转AI）
- **README.md**: 项目文档、快速开始、对比表
- **错误处理与降级策略**: 空池、超时、跑题、发言过短、议题无法解构
- **快速入口**: `快速圆桌：人物A+人物B：议题` 跳过人物推荐步骤
- **描述字段清洗规则**: 从 perspective skill 的 description 中智能提取有用信息

### Changed
- 子代理提示词模板从 SKILL.md 内联移至 PROMPTS.md
- PERSONAS.md 从静态人物表重构为冲突配对参考 + 选人原则
- 发言排序从随机改为按逻辑顺序（核心张力 → 对立 → 新维度）
- 发言顺序规则从自由改为结构化（弱者先 → 强者后）
- 第二决策节点触发条件从「任一」收紧为「同时满足两个以上」

### Fixed
- 修复人物池为空时的死胡同问题
- 修复子代理发言不回应具体变量/假设的问题（加入纠偏指令）
- 修复整合方案偏向某个参会者的问题（三方批评自检）

## [1.0.0] — 2026-05-09

### Added
- 八步圆桌会议流程：动态发现 → 议题解构 → 推荐参会者 → 开场陈述 → 深度探索 → 交叉讨论 → 决策节点 → 整合方案
- 动态人物池扫描（`ls ~/.claude/skills/*-perspective`）
- 用户干预机制（随时插入、中途加人、单独回应）
- 质量自检清单
- PERSONAS.md 人物库
