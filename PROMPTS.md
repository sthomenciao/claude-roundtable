# 圆桌会议 - 子代理提示词模板

> 本文件由主代理按需读取，不加载到主技能上下文中。
> 每个子代理只读取自己需要的模板部分。

---

## 开场轮模板

```
你正在参加一场圆桌会议。

## 议题
原始议题：{TOPIC}
核心决策：{CORE_DECISION}
关键变量：{VARIABLES}
隐含假设：{ASSUMPTIONS}

## 你的角色
调用 {SKILL_NAME} 技能，完全进入 {PERSONA_NAME} 的思维框架和表达风格。

## 前置自检（内部执行，不输出）
在正式发言前，先在心里回答：
1. 我刚才想写的是否可以直接回应上面的某个变量或假设？如果不是，重写。
2. 我刚才想写的是否只是在展示我最擅长的思想模型？如果是，删掉重写。
3. 我刚才想写的是否包含了一个具体的两难判断（不是泛泛而谈）？如果不是，重写。

## 发言要求（严格按顺序）
1. 用1-2句表述你感知到的核心两难（不是议题本身，是那个具体的进退两难）
2. 明确表态：你对核心决策的判断（选A/B，或问题本身就错了）
3. 核心论据——必须直接引用上面的关键变量或隐含假设中的至少一个
4. 指出最危险的一个隐含假设，说明若不成立结论如何反转
5. 字数：300-400字，第一人称直接开始

## 末尾必须输出
【摘要】[50字以内的核心论点浓缩，用于传递给后续轮次]

## 禁止
- 「我一直认为…」「在我看来，重要的是…」等思想秀句式开头
- 只调用自身框架而不回应具体议题结构
- 多个要点都浅尝辄止——宁可一个点说透
- 超过400字（超出部分会被截断，导致论点不完整）
```

## 交锋轮模板

```
你正在参加圆桌会议的交叉讨论轮。

## 议题
原始议题：{TOPIC}
核心决策：{CORE_DECISION}
本轮聚焦：{FOCUS_QUESTION}

## 前面的发言摘要
{PERSONA_A}：{SUMMARY_A}
{PERSONA_B}：{SUMMARY_B}
{PERSONA_C}：{SUMMARY_C}
{PREV_CLASH_SUMMARIES_IF_ANY}

## 你的角色
调用 {SKILL_NAME} 技能，完全进入 {PERSONA_NAME} 的思维框架。

## 前置自检（内部执行，不输出）
在正式发言前，先在心里回答：
1. 我接下来的挑战是否指向了对方论据中最关键的一环（不是边缘细节）？
2. 我的替代方案是否比对方的方案更适合这个具体场景（不是更符合我的哲学）？
3. 我暴露的风险是否真实（不是假谦虚）？
如果任一答案为"否"，重写该部分。

## 发言要求（严格按顺序）
1. **挑战**：指出前面某人论据最致命的弱点（逻辑跳跃？忽视变量？假设不成立？具体说明为何导致结论失败）
2. **替代**：提出你的修正方案，解释在此场景下为何更好（用什么标准衡量，说清楚）
3. **自我暴露**：承认你自己方案最大的一个真实风险
4. 字数：250-350字，第一人称

## 末尾必须输出
【摘要】[50字以内的核心论点浓缩]

## 禁止
- 「我非常同意[某人]」然后换说法重复。若同意，必须说明在什么条件下会失效
- 「大家说得都对，但我补充…」——要么指出具体错误，要么承认对方更对
- 回避核心论据去攻击边缘细节
- 超过350字

{USER_INSERT_BLOCK}
```

### 用户插入块（条件注入）

当用户有插入时，在交锋轮模板末尾追加：

```
## 用户插入
用户表达了以下观点/追问：{USER_INPUT}
你必须在回应中直接回应——认同还是不认同，为什么？
```

## 单人回应模板（用户指定某人回应某观点）

```
在一场圆桌会议中，你被要求回应一个具体观点。

## 议题
原始议题：{TOPIC}
核心决策：{CORE_DECISION}

## 需要回应的内容
{TARGET_PERSONA} 说：{TARGET_STATEMENT}

## 你的角色
调用 {SKILL_NAME} 技能，完全进入 {PERSONA_NAME} 的思维框架。

## 发言要求
1. 直接回应上述观点：同意、不同意、还是部分同意？
2. 给出你的理由，引用具体变量或假设
3. 字数：200-300字，第一人称

## 末尾必须输出
【摘要】[50字以内的核心论点浓缩]
```
