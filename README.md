# Roundtable — AI 圆桌会议

让多位历史/思想人物的 AI 分身围绕你的真实议题展开**有实质交锋**的多轮讨论，最终给出整合方案。

不是各说各话的表演秀，而是**互相挑战、互相追问、互相修正**的真实辩论。

---

## 前置条件

- [Claude Code](https://claude.ai/code) 或支持 Skill 工具的 Claude 环境
- 至少一个 `*-perspective` skill（参会者）

### 安装参会者

```bash
# 克隆示例人物库到 skills 目录
git clone https://github.com/sthomenciao/roundtable-personas ~/.claude/skills/

# 或手动安装单个 perspective skill
cp -r my-feynman-skill ~/.claude/skills/feynman-perspective
```

安装后，下次启动圆桌自动可用。

## 快速开始

### 方式一：完整流程（推荐首次使用）

```
用户：开个圆桌

Claude：圆桌会议已就绪。当前可用参会者：费曼、马斯克、张雪峰…
       请告诉我你想讨论的主题或问题。

用户：35岁中台高级经理，要不要花一年在职转AI工程师？

[进入议题解构 → 推荐参会者 → 开场陈述 → 交叉交锋 → 整合方案]
```

### 方式二：快速入口（已知要谁参与）

```
用户：快速圆桌：费曼+马斯克+张雪峰：35岁中台经理转AI岗的决策分析

[跳过人物推荐，直接进入议题解构和讨论]
```

### 方式三：随时干预

| 你说 | 效果 |
|------|------|
| "让马斯克回应费曼的成本假设" | 单独让马斯克正面回应 |
| "加入毛泽东" | 中途引入新参会者，以迟到者角度指出遗漏 |
| "我同意张雪峰的市场判断" | 其他参会者针对这一点做最后一轮回应 |
| "跳到整合" | 跳过剩余讨论，直接输出最终方案 |
| "补充：我其实有6个月存款" | 更新议题解构中的变量，所有人重新校准 |

## 核心机制

```
动态发现 → 议题解构 → 锁定发言 → 对抗交锋 → 整合方案
    ↑                                    ↓
    └──────────── 用户随时干预 ←────────┘
```

**议题解构**：把模糊议题拆成"核心决策 + 关键变量 + 隐含假设"，让后续讨论有锚点。

**锁定发言**：每位参会者必须直接引用具体变量或假设，禁止泛泛而谈。

**对抗交锋**：每人必须指出他人论据最致命的弱点，提出替代方案，并暴露自己的真实风险。

**整合超越立场**：最终方案不是任何人的原始立场，而是经过碰撞后的综合判断，并通过"三方批评自检"验证独立性。

## 新增人物

创建一个 `*-perspective` skill 目录，内含 `SKILL.md`：

```yaml
---
name: your-persona-name
description: |
  [人物]的思维框架。基于[来源]提炼。
  擅长：[核心能力1]、[核心能力2]。
  用途：作为思维顾问，分析[领域]问题。
---

# [人物名]的思维框架

## 核心追问
面对任何议题时，第一个追问的问题是什么？

## 思维模型
...

## 表达风格
...
```

安装到 `~/.claude/skills/your-persona-perspective/`，下次圆桌自动发现。

## 为什么不是普通的"多角色对话"

| | 普通多角色对话 | Roundtable |
|--|---------------|-----------|
| 议题处理 | 直接开始 | **先解构为变量+假设**，有锚点 |
| 发言约束 | 自由发挥 | **必须引用具体变量**，禁止泛泛而谈 |
| 互动模式 | 各说各话 | **互相挑战**，指出对方最致命弱点 |
| 信息传递 | 传递全文 | **自产50字摘要**，不膨胀上下文 |
| 最终方案 | 简单汇总 | **超越任何单一立场**，经三方批评自检 |
