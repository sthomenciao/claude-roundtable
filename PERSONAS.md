# 圆桌人物库

> 此文件仅作参考索引。实际人物池以每次启动时自动扫描 `~/.claude/skills/*-perspective` 的结果为准。

## 参会者选择原则

1. **冲突优先**：选择思维框架有结构性冲突的组合，冲突产生洞察
2. **挑战假设**：至少一位会直接攻击议题的隐含假设
3. **议题相关**：核心领域必须与议题关键变量有实质交集
4. **规模控制**：3人最聚焦，最多4人，超过4人会发散

## 典型冲突配对

以下描述了几种常见的思维框架冲突，帮助主代理在动态人物池中识别有价值的配对：

| 冲突类型 | 说明 | 适合议题 |
|----------|------|---------|
| 理想主义 vs 实用主义 | 一方追求本质/原理，一方看市场/数据 | 职业选择、学习方法 |
| 激进路径 vs 渐进路径 | 一方主张大幅重构，一方主张小步迭代 | 创业、技术决策 |
| 用户导向 vs 工程导向 | 一方从体验出发，一方从可行性出发 | 产品设计 |
| 矛盾分析 vs 成本分析 | 一方识别力量对比，一方拆解资源结构 | 战略竞争 |
| 技术乐观 vs 技术现实 | 一方看趋势，一方看当前可靠性 | AI/技术趋势 |

## 新增人物 Skill 建议

为让圆桌交锋更深入，建议每个 perspective skill 的 SKILL.md frontmatter 或正文包含：

```yaml
# 建议在 description 末尾或正文开头明确：
core_question: 面对任何议题时第一个追问的问题
blind_spot: 惯性盲点（圆桌中其他人可攻击这里）
clash_with: 最容易与谁产生分歧，为什么
```

新增人物只需创建 `*-perspective` skill，下次开圆桌自动可用。
